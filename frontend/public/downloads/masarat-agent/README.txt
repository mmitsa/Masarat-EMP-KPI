  وكيل مزامنة أجهزة البصمة — Masarat Biometric Agent v1.0
══════════════════════════════════════════════════════════════════

  الوصف:
  ──────
  برنامج خفيف يعمل على شبكة العميل المحلية، يقرأ سجلات الحضور
  من أجهزة ZKTeco عبر بروتوكول UDP/TCP ثم يرسلها تلقائياً
  إلى منصة مسارات السحابية.


  المتطلبات:
  ──────────
  • Node.js 18 أو أحدث (https://nodejs.org)
  • جهاز كمبيوتر على نفس الشبكة المحلية لأجهزة البصمة
  • اتصال إنترنت للتواصل مع السيرفر السحابي


  خطوات التشغيل:
  ────────────────

  1. ثبّت Node.js على الجهاز المحلي (إن لم يكن مثبتاً):
     https://nodejs.org → حمّل LTS → ثبّته

  2. أضف جهاز بصمة في المنصة:
     • ادخل على: إعدادات البصمة → إدارة الأجهزة → إضافة جهاز
     • اختر وضع الاتصال: "وكيل محلي"
     • أدخل الرقم التسلسلي وباقي البيانات
     • بعد الحفظ: انسخ مفتاح API الذي يظهر

  3. عدّل ملف config.json:
     • serverUrl: عنوان المنصة (مثل https://unified.mmit.sa)
     • apiKey: الصق مفتاح API المنسوخ من الخطوة 2
     • devices: أضف أجهزة البصمة على الشبكة المحلية:
       - ip: عنوان IP للجهاز (مثل 192.168.1.201)
       - port: المنفذ (عادة 4370)
       - protocol: البروتوكول (udp أو tcp)
       - name: اسم تعريفي للجهاز
     • syncIntervalMinutes: فترة المزامنة بالدقائق (الافتراضي: 5)

  4. شغّل الوكيل:
     افتح Command Prompt في مجلد الوكيل واكتب:

       node masarat-agent.js

     ستظهر رسائل المزامنة في الشاشة.

  5. (اختياري) تثبيت كخدمة Windows:
     • حمّل NSSM من https://nssm.cc/download
     • ضع nssm.exe في مجلد مضاف لـ PATH (مثل C:\Windows)
     • شغّل install-service.bat كمسؤول (Run as Administrator)
     • الخدمة ستعمل تلقائياً عند تشغيل الكمبيوتر


  هيكل الملفات:
  ──────────────
  masarat-agent.js     — البرنامج الرئيسي
  config.json          — ملف الإعدادات (عدّله قبل التشغيل)
  install-service.bat  — تثبيت كخدمة Windows
  agent.log            — سجل العمليات (يُنشأ تلقائياً)
  README.txt           — هذا الملف


  استكشاف الأخطاء:
  ─────────────────
  • "فشل الاتصال بالجهاز":
    - تأكد أن IP الجهاز صحيح وأنه على نفس الشبكة
    - تأكد أن المنفذ 4370 مفتوح
    - جرّب تغيير البروتوكول من udp إلى tcp

  • "خطأ في الاتصال بالسيرفر":
    - تأكد أن serverUrl صحيح
    - تأكد أن الجهاز متصل بالإنترنت
    - تأكد أن apiKey صحيح

  • "مفتاح API غير صالح":
    - تأكد من نسخ المفتاح كاملاً من صفحة إعدادات البصمة
    - تأكد أن الجهاز مسجل بوضع "وكيل محلي" وليس "سحب" أو "دفع"


  الدعم الفني:
  ─────────────
  للمساعدة تواصل مع فريق الدعم الفني لمنصة مسارات.
